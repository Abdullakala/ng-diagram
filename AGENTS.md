# أنواع الوكلاء المتوفرة في المشروع (Available Agents)

يحتوي مشروع **ng-diagram** على نوعين رئيسيين من الوكلاء يعملان بشكل متكامل لتوفير تجربة تفاعلية ذكية للتحكم بالمخططات.

---

## 1. وكيل الذكاء الاصطناعي (AI Development Agent)

**الموقع:** `apps/angular-demo/src/app/ai-agent/`

وكيل تفاعلي يتيح للمستخدم التحكم بالمخطط من خلال **أوامر نصية باللغة العربية والإنجليزية**.

### الملفات

| الملف | الوصف |
|-------|-------|
| `ai-agent-panel.component.ts` | مكوّن واجهة المستخدم للوحة الوكيل |
| `ai-agent-panel.component.html` | قالب العرض |
| `ai-agent-panel.component.scss` | التنسيقات |
| `command-processor.service.ts` | خدمة معالجة الأوامر |

### الوظائف والأوامر المدعومة

#### 🔨 إنشاء العقد (Create)

| الأمر | الوصف |
|-------|-------|
| `add node "Label" [type:chip] [at 100,200]` | إنشاء عقدة جديدة مع تحديد الموضع والنوع |
| `اضف عقدة "التسمية"` | الأمر العربي المكافئ |

#### 🔗 ربط العقد (Connect)

| الأمر | الوصف |
|-------|-------|
| `connect "Node A" to "Node B"` | ربط عقدتين ببعضهما |
| `link "Node A" to "Node B"` | بديل للأمر connect |
| `وصل "عقدة أ" الى "عقدة ب"` | الأمر العربي المكافئ |

#### 🗑️ حذف العقد (Delete)

| الأمر | الوصف |
|-------|-------|
| `delete "Node A"` | حذف عقدة محددة |
| `delete selected` | حذف العناصر المحددة |
| `احذف "عقدة"` أو `ازل "عقدة"` | الأمر العربي المكافئ |

#### 🔍 البحث (Search)

| الأمر | الوصف |
|-------|-------|
| `search "keyword"` أو `find "text"` | البحث عن عقد أو حواف |
| `ابحث "كلمة"` | البحث بالعربي |

#### 📋 عرض القوائم (List)

| الأمر | الوصف |
|-------|-------|
| `list nodes` | عرض جميع العقد |
| `list edges` | عرض جميع الحواف |
| `اعرض العقد` أو `اعرض الحواف` | الأمر العربي المكافئ |

#### 🎯 التحديد (Select)

| الأمر | الوصف |
|-------|-------|
| `select "Node A"` | تحديد عقدة معينة |
| `select all` | تحديد الكل |
| `deselect` | إلغاء التحديد |
| `اختر` أو `حدد` | الأمر العربي المكافئ |

#### 🔍 التكبير والتصغير (Zoom)

| الأمر | الوصف |
|-------|-------|
| `zoom in` | تكبير |
| `zoom out` | تصغير |
| `zoom fit` | ملاءمة المحتوى |

#### 📍 نقل العقد (Move)

| الأمر | الوصف |
|-------|-------|
| `move "Node A" to 200, 300` | نقل عقدة إلى موضع محدد |
| `انقل "عقدة" الى 200, 300` | الأمر العربي المكافئ |

#### 📊 بناء سير العمل (Build Workflow)

| الأمر | الوصف |
|-------|-------|
| `build workflow: Start, Process, Review, End` | بناء سير عمل من عدة خطوات |
| `بناء سير عمل: بداية, معالجة, مراجعة, نهاية` | الأمر العربي المكافئ |

#### ℹ️ معلومات العقدة (Info)

| الأمر | الوصف |
|-------|-------|
| `info "Node A"` | عرض معلومات تفصيلية عن عقدة |

#### 🧹 أوامر مساعدة (Utility)

| الأمر | الوصف |
|-------|-------|
| `clear all` | مسح المخطط بالكامل |
| `help` أو `مساعدة` | عرض المساعدة |
| `?` | مساعدة سريعة |

### مميزات الوكيل

- **دعم ثنائي اللغة** (العربية والإنجليزية)
- **تحليل مرن** للأوامر النصية
- **عمليات دفعية** لإنشاء سير العمل
- **إدراك السياق** عبر استخدام العناصر المحددة
- **رسائل تغذية راجعة** مع رموز تعبيرية
- **اقتراحات سريعة** عبر أزرار جاهزة

## 2. نظام معالجة الأوامر الأساسي (Core Command Handler)

**الموقع:** `packages/ng-diagram/projects/ng-diagram/src/core/src/command-handler/`

نظام داخلي لإدارة عمليات المخطط وتحولات الحالة من خلال **نمط الأوامر (Command Pattern)** مع دعم الأنواع الآمنة.

### الملفات

| الملف | الوصف |
|-------|-------|
| `command-handler.ts` | تنفيذ معالج الأوامر المركزي |
| `command-handler.interface.ts` | تعريفات الأنواع والواجهات |
| `commands/` | مجلد يحتوي على جميع الأوامر المتوفرة |

### أوامر النموذج (Model Commands)

| الأمر | الوظيفة |
|-------|---------|
| `AddNodesCommand` | إضافة عقد جديدة للمخطط |
| `UpdateNodeCommand` / `UpdateNodesCommand` | تحديث خصائص العقد |
| `DeleteNodesCommand` | حذف العقد |
| `AddEdgesCommand` | إضافة حواف (روابط) |
| `UpdateEdgesCommand` | تحديث الحواف |
| `DeleteEdgesCommand` | حذف الحواف |
| `AddEdgeLabelsCommand` | إضافة تسميات للحواف |
| `UpdateEdgeLabelsCommand` | تحديث تسميات الحواف |
| `DeleteEdgeLabelsCommand` | حذف تسميات الحواف |
| `AddPortsCommand` / `AddPortsBulkCommand` | إضافة منافذ |
| `UpdatePortsCommand` | تحديث المنافذ |
| `DeletePortsCommand` | حذف المنافذ |
| `ClearModelCommand` | مسح المخطط بالكامل |
| `PaletteDropNodeCommand` | إسقاط عقدة من لوحة الأدوات |

### أوامر التحديد (Selection Commands)

| الأمر | الوظيفة |
|-------|---------|
| `SelectCommand` | تحديد عنصر |
| `SelectAllCommand` | تحديد الكل |
| `DeselectCommand` | إلغاء تحديد عنصر |
| `DeselectAllCommand` | إلغاء تحديد الكل |
| `SelectEndCommand` | إنهاء عملية التحديد |
| `DeleteSelectionCommand` | حذف العناصر المحددة |

### أوامر العرض (Viewport Commands)

| الأمر | الوظيفة |
|-------|---------|
| `ZoomCommand` | تكبير / تصغير بنسبة محددة |
| `ZoomToFitCommand` | ملاءمة المحتوى |
| `MoveViewportCommand` / `MoveViewportByCommand` | تحريك منطقة العرض |
| `CenterOnNodeCommand` | توسيط العرض على عقدة |
| `CenterOnRectCommand` | توسيط العرض على منطقة |

### أوامر التحويل (Transform Commands)

| الأمر | الوظيفة |
|-------|---------|
| `MoveNodesByCommand` | سحب ونقل العقد |
| `MoveNodesStartCommand` / `MoveNodesStopCommand` | بداية ونهاية عملية النقل |
| `ResizeNodeCommand` | تغيير حجم العقدة |
| `ResizeNodeStartCommand` / `ResizeNodeStopCommand` | بداية ونهاية تغيير الحجم |
| `RotateNodeToCommand` | تدوير العقدة |
| `RotateNodeStartCommand` / `RotateNodeStopCommand` | بداية ونهاية التدوير |

### أوامر مساعدة (Utility Commands)

| الأمر | الوظيفة |
|-------|---------|
| `CopyCommand` / `CutCommand` / `PasteCommand` | عمليات الحافظة |
| `BringToFrontCommand` / `SendToBackCommand` | ترتيب الطبقات |
| `HighlightGroupCommand` / `HighlightGroupClearCommand` | تمييز المجموعات |
| `AddToGroupCommand` / `RemoveFromGroupCommand` | إدارة المجموعات |
| `StartLinkingCommand` / `MoveTemporaryEdgeCommand` / `FinishLinkingCommand` | رسم الحواف |
| `InitCommand` | تهيئة المخطط |

## 3. معالجات الدُفعات (Batch Processors)

وكلاء مساعدون لتحسين الأداء عبر تجميع العمليات المتكررة.

### FlowResizeBatchProcessorService

**الموقع:** ضمن نظام الأوامر الأساسي

| الوظيفة | الوصف |
|---------|-------|
| تجميع أحداث `ResizeObserver` | تحسين الأداء عند تغيير أحجام متعددة |
| تصنيف حسب النوع | منافذ، عقد، تسميات حواف |
| إزالة التكرار | تجاهل التغييرات الأقل من 1 بكسل |

### PortBatchProcessor

**الموقع:** `packages/ng-diagram/projects/ng-diagram/src/core/src/port-batch-processor/`

| الوظيفة | الوصف |
|---------|-------|
| تجميع تحديثات المنافذ | تحديث الحجم والموضع دفعة واحدة |
| دعم التجميع لكل عقدة | تحسين الأداء |
| جدولة عبر `queueMicrotask` | ضمان الكفاءة |

### LabelBatchProcessor

**الموقع:** `packages/ng-diagram/projects/ng-diagram/src/core/src/label-batch-processor/`

| الوظيفة | الوصف |
|---------|-------|
| تجميع عمليات التسميات | منع التعارضات أثناء التحديثات المتزامنة |
| استراتيجيات تفريغ متعددة | للإضافات والتحديثات |

## 5. مقارنة بين الوكلاء

| الجانب | وكيل الذكاء الاصطناعي | نظام الأوامر الأساسي | معالجات الدُفعات |
|--------|----------------------|---------------------|-----------------|
| **الموقع** | تطبيق العرض التجريبي | مكتبة ng-diagram الأساسية | مكتبة ng-diagram الأساسية |
| **الغرض** | واجهة تفاعلية للمستخدم | إدارة الحالة الداخلية | تحسين الأداء |
| **الواجهة** | نص طبيعي (عربي/إنجليزي) | واجهات برمجية مُنمّطة | داخلية تلقائية |
| **الاتجاه** | أحادي (مستخدم ← تطبيق) | ثنائي (أحداث واستجابات) | أحادي (تجميع ← تنفيذ) |
| **دعم اللغة العربية** | ✅ نعم | ❌ لا | ❌ لا |
